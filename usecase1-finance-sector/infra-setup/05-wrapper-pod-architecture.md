# Wrapper Pod Architecture

This guide covers the dynamically created wrapper pods that serve graph queries.

## Overview

Wrapper pods are ephemeral Kubernetes pods created by the Control Plane to serve graph instances. Each wrapper:
- Loads graph data from GCS into memory
- Serves Cypher/GQL queries via HTTP
- Is automatically created/destroyed based on instance lifecycle

## Wrapper Types

| Type | Engine | Use Case |
|------|--------|----------|
| `ryugraph` | Ryugraph | High-performance graph queries, algorithms |
| `falkordb` | FalkorDB (Redis-based) | Smaller graphs, Redis compatibility |

## 1. Pod Specification

The Control Plane dynamically generates pod specs. Here's an example:

```yaml
# wrapper-pod-example.yaml (generated by control-plane)
apiVersion: v1
kind: Pod
metadata:
  name: wrapper-abc123def
  namespace: graph-olap
  labels:
    app: ryugraph-wrapper
    wrapper-type: ryugraph
    instance-id: "42"
    url-slug: abc123def
    snapshot-id: "100"
    owner-email: user-at-example-com
spec:
  restartPolicy: Never
  serviceAccountName: wrapper

  containers:
    - name: wrapper
      image: gcr.io/YOUR_PROJECT/ryugraph-wrapper:v1.0.0
      imagePullPolicy: IfNotPresent

      ports:
        - containerPort: 8000
          name: http

      env:
        - name: WRAPPER_INSTANCE_ID
          value: "42"
        - name: WRAPPER_URL_SLUG
          value: "abc123def"
        - name: WRAPPER_SNAPSHOT_ID
          value: "100"
        - name: WRAPPER_MAPPING_ID
          value: "10"
        - name: WRAPPER_MAPPING_VERSION
          value: "1"
        - name: WRAPPER_OWNER_ID
          value: "user@example.com"
        - name: WRAPPER_GCS_BASE_PATH
          value: "gs://your-bucket/snapshots/100"
        - name: WRAPPER_CONTROL_PLANE_URL
          value: "http://control-plane:8080"
        - name: WRAPPER_INSTANCE_URL
          value: "http://wrapper-abc123def.graph-olap.svc.cluster.local:8000"
        - name: DATABASE_PATH
          value: "/data/db"
        - name: LOG_LEVEL
          value: "INFO"
        - name: LOG_FORMAT
          value: "json"
        - name: GRAPH_OLAP_INTERNAL_API_KEY
          value: "internal-api-key"

      volumeMounts:
        - name: data
          mountPath: /data

      resources:
        requests:
          memory: "2Gi"
          cpu: "1"
        limits:
          memory: "4Gi"
          cpu: "2"

      # Startup probe - data loading can take 60-150s
      startupProbe:
        httpGet:
          path: /health
          port: 8000
        initialDelaySeconds: 5
        periodSeconds: 5
        timeoutSeconds: 3
        failureThreshold: 30  # 150s max startup

      # Readiness probe - fast checks after startup
      readinessProbe:
        httpGet:
          path: /ready
          port: 8000
        periodSeconds: 2
        timeoutSeconds: 1
        failureThreshold: 3

      # Liveness probe
      livenessProbe:
        httpGet:
          path: /health
          port: 8000
        initialDelaySeconds: 10
        periodSeconds: 10
        timeoutSeconds: 5
        failureThreshold: 3

  volumes:
    - name: data
      emptyDir: {}
```

## 2. Service (Per-Wrapper)

Each wrapper gets its own ClusterIP service:

```yaml
# wrapper-service-example.yaml (generated by control-plane)
apiVersion: v1
kind: Service
metadata:
  name: wrapper-abc123def
  namespace: graph-olap
  labels:
    app: ryugraph-wrapper
    instance-id: "42"
    url-slug: abc123def
    wrapper-type: ryugraph
  annotations:
    # Enable container-native load balancing (NEG)
    cloud.google.com/neg: '{"ingress": true}'
spec:
  selector:
    app: ryugraph-wrapper
    url-slug: abc123def
  ports:
    - port: 8000
      targetPort: 8000
      name: http
  type: ClusterIP
```

## 3. Dynamic Resource Sizing

The Control Plane calculates resources based on snapshot size:

```python
# Resource calculation logic (simplified)
def calculate_resources(snapshot_size_bytes: int) -> dict:
    size_gb = snapshot_size_bytes / (1024 ** 3)

    if size_gb < 1:
        return {
            "memory_request": "1Gi",
            "memory_limit": "2Gi",
            "cpu_request": "500m",
            "cpu_limit": "1",
        }
    elif size_gb < 5:
        return {
            "memory_request": "2Gi",
            "memory_limit": "4Gi",
            "cpu_request": "1",
            "cpu_limit": "2",
        }
    elif size_gb < 20:
        return {
            "memory_request": "8Gi",
            "memory_limit": "16Gi",
            "cpu_request": "2",
            "cpu_limit": "4",
        }
    else:
        return {
            "memory_request": "16Gi",
            "memory_limit": "32Gi",
            "cpu_request": "4",
            "cpu_limit": "8",
        }
```

## 5. Health Check Endpoints

Each wrapper exposes:

| Endpoint | Purpose | When 200 |
|----------|---------|----------|
| `/health` | Liveness | FastAPI is running |
| `/ready` | Readiness | Graph data loaded, ready for queries |

## 6. Data Loading Process

```
┌─────────────────────────────────────────────────────────────┐
│                   Wrapper Startup                            │
└─────────────────────────────────────────────────────────────┘

1. Pod starts
   │
   ▼
2. Read environment variables (GCS path, snapshot ID, etc.)
   │
   ▼
3. Download snapshot from GCS
   │  └── gs://bucket/snapshots/{snapshot_id}/
   │      ├── nodes.parquet
   │      ├── edges.parquet
   │      └── metadata.json
   │
   ▼
4. Load data into graph engine
   │  └── Parse parquet files
   │  └── Build in-memory graph structure
   │  └── Create indexes
   │
   ▼
5. /ready returns 200
   │
   ▼
6. Kubernetes adds pod to Service endpoints
   │
   ▼
7. Ready for queries
```

## 7. Monitoring Labels

All wrapper resources include labels for monitoring and cleanup:

| Label | Purpose |
|-------|---------|
| `app` | `{wrapper-type}-wrapper` (e.g., `ryugraph-wrapper`) |
| `wrapper-type` | `ryugraph` or `falkordb` |
| `instance-id` | Database instance ID |
| `url-slug` | UUID for URL routing |
| `snapshot-id` | Source snapshot ID |
| `owner-email` | Sanitized owner email (for cleanup queries) |

## 8. Cleanup by Owner

List all wrappers for a user:

```bash
kubectl get pods -n graph-olap \
    -l owner-email=user-at-example-com \
    -o wide
```

Delete all wrappers for a user:

```bash
kubectl delete pods -n graph-olap \
    -l owner-email=user-at-example-com
```
